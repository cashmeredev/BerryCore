The gcc-arm 9.3.0 port has been removed from BerryCore due to
compatibility issues. Run this script to clean up your Passport:


STEP 1: Run Cleanup Script
───────────────────────────────────────────────────────────────
sh REMOVE_GCC_ARM.sh

This will:
  - Remove GCC 9.3.0 binaries
  - Remove corrupted specs file  
  - Remove GCC 9.3.0 libraries
  - Restore working GCC 4.6.3


STEP 2: Verify GCC Works
───────────────────────────────────────────────────────────────
gcc --version
# Should show: gcc (GCC) 4.6.3

sh QUICK_TEST.sh
# Should pass all tests


STEP 3: Update BerryCore (Optional)
───────────────────────────────────────────────────────────────
# Pull latest with gcc-arm removed
qpkg update

WHAT YOU CAN DO NOW
───────────────────────────────────────────────────────────────

✓ Compile C programs with GCC 4.6.3
✓ Compile C++ programs with G++ 4.6.3
✓ Use all other BerryCore tools normally

The working GCC 4.6.3 is perfect for:
  - Building utilities
  - Compiling scripts
  - Learning C/C++
  - Most development tasks

TRY BERRYSNIP!

ISSUES TO REPORT
───────────────────────────────────────────────────────────────

If the cleanup script has problems, manually remove:

rm -f $NATIVE_TOOLS/bin/gcc
rm -f $NATIVE_TOOLS/bin/gcc.real
rm -f $NATIVE_TOOLS/bin/g++
rm -f $NATIVE_TOOLS/bin/g++.real
rm -rf $NATIVE_TOOLS/lib/gcc/arm-blackberry-qnx8eabi/9.3.0
rm -rf $NATIVE_TOOLS/libexec/gcc/arm-blackberry-qnx8eabi/9.3.0

Then source environment:
source $NATIVE_TOOLS/env.sh
